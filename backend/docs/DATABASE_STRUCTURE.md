# Структура базы данных Collide

Документ описывает структуру PostgreSQL-базы данных для проекта **Collide** — веб-приложения для коллекционеров.

Схема рассчитана на функционал:

- регистрация и профиль пользователей;
- подписки на пользователей;
- категории коллекций и предметов;
- коллекции;
- коллекционные предметы;
- лайки предметов;
- оценки коллекций по 5-балльной системе;
- избранные предметы и коллекции;
- комментарии к предметам и коллекциям;
- личные чаты между двумя пользователями;
- сообщения в личных чатах;
- статистика профиля и коллекций.

---

## 1. Общая ER-структура

```text
app_users
 ├── user_follows
 ├── collections
 │    ├── items
 │    ├── collection_ratings
 │    └── collection_comments
 ├── items
 │    ├── item_likes
 │    └── item_comments
 ├── favorite_items
 ├── favorite_collections
 ├── chats.first_user_id
 ├── chats.second_user_id
 └── messages.sender_id

categories
 ├── collections
 └── items

chats
 └── messages
```

## 4. Таблицы

## `app_users`

Хранит пользователей приложения.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `UUID` | Уникальный идентификатор пользователя |
| `email` | `VARCHAR(254)` | Email пользователя |
| `username` | `VARCHAR(50)` | Уникальный логин |
| `password_hash` | `VARCHAR(255)` | Хэш пароля |
| `display_name` | `VARCHAR(80)` | Отображаемое имя |
| `bio` | `VARCHAR(500)` | Описание профиля |
| `avatar_url` | `TEXT` | Ссылка на аватар |
| `status_message` | `VARCHAR(120)` | Статус пользователя |
| `role` | `VARCHAR(20)` | Роль пользователя: `USER` или `ADMIN` |
| `is_enabled` | `BOOLEAN` | Активен ли аккаунт |
| `last_seen_at` | `TIMESTAMPTZ` | Время последнего посещения |
| `created_at` | `TIMESTAMPTZ` | Дата создания |
| `updated_at` | `TIMESTAMPTZ` | Дата обновления |

Особенности:

- `email` уникален без учёта регистра.
- `username` уникален без учёта регистра.
- пароль хранится только в виде хэша, не открытым текстом.

---

## `user_follows`

Хранит подписки пользователей друг на друга.

| Поле | Тип | Описание |
|---|---|---|
| `follower_id` | `UUID` | Кто подписался |
| `following_id` | `UUID` | На кого подписался |
| `created_at` | `TIMESTAMPTZ` | Дата подписки |

Ключи и ограничения:

- `PRIMARY KEY (follower_id, following_id)` не позволяет повторно подписаться на одного пользователя.
- `CHECK (follower_id <> following_id)` запрещает подписку на самого себя.
- при удалении пользователя связанные подписки удаляются автоматически.

| Поле | Тип | Описание |
|---|---|---|

Категории по умолчанию:

| `slug` | Название |
|---|---|

| `records` | Пластинки |
| `books` | Книги |
| `watches` | Часы |

## `collections`

Хранит коллекции пользователей.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `UUID` | Уникальный идентификатор коллекции |
| `owner_id` | `UUID` | Владелец коллекции |
| `category_id` | `UUID` | Категория коллекции |
| `title` | `VARCHAR(120)` | Название коллекции |
| `description` | `TEXT` | Описание коллекции |
| `cover_image_url` | `TEXT` | Ссылка на обложку |
| `visibility` | `VARCHAR(20)` | Видимость: `PUBLIC` или `PRIVATE` |
| `created_at` | `TIMESTAMPTZ` | Дата создания |
| `updated_at` | `TIMESTAMPTZ` | Дата обновления |

Особенности:

- при удалении пользователя его коллекции удаляются;
- при удалении категории поле `category_id` становится `NULL`;
- общая стоимость коллекции не хранится напрямую, а считается по предметам через `collection_stats`.

---

## `items`

Хранит коллекционные предметы.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `UUID` | Уникальный идентификатор предмета |
| `owner_id` | `UUID` | Владелец предмета |
| `collection_id` | `UUID` | Коллекция предмета, может быть `NULL` |
| `category_id` | `UUID` | Категория предмета |
| `title` | `VARCHAR(120)` | Название предмета |
| `short_description` | `VARCHAR(60)` | Краткое описание для карточки |
| `full_description` | `TEXT` | Полное описание |
| `image_url` | `TEXT` | Ссылка на изображение |
| `price_amount` | `NUMERIC(12,2)` | Цена |
| `currency` | `CHAR(3)` | Валюта: `RUB`, `USD`, `EUR` |
| `status` | `VARCHAR(30)` | Статус предмета |
| `visibility` | `VARCHAR(20)` | Видимость |
| `created_at` | `TIMESTAMPTZ` | Дата создания |
| `updated_at` | `TIMESTAMPTZ` | Дата обновления |

Допустимые статусы:

| Значение | Значение для интерфейса |
|---|---|
| `IN_COLLECTION` | В коллекции |
| `FOR_SALE` | В продаже |
| `FOR_EXCHANGE` | Для обмена |
| `ARCHIVED` | В архиве |

Особенности:

- `collection_id` может быть `NULL`, если предмет добавлен в общий список без выбора коллекции;
- цена может быть пустой, но не может быть отрицательной;
- краткое описание ограничено 60 символами.

## `item_likes`

Хранит лайки предметов.

| Поле | Тип | Описание |
|---|---|---|
| `item_id` | `UUID` | Предмет |
| `user_id` | `UUID` | Пользователь, поставивший лайк |
| `created_at` | `TIMESTAMPTZ` | Дата лайка |

Ключ:

```sql
PRIMARY KEY (item_id, user_id)
```

Один пользователь может поставить одному предмету только один лайк.

---

## `collection_ratings`

Хранит оценки коллекций по 5-балльной системе.

| Поле | Тип | Описание |
|---|---|---|
| `collection_id` | `UUID` | Коллекция |
| `user_id` | `UUID` | Пользователь, поставивший оценку |
| `rating` | `SMALLINT` | Оценка от 1 до 5 |
| `created_at` | `TIMESTAMPTZ` | Дата создания оценки |
| `updated_at` | `TIMESTAMPTZ` | Дата изменения оценки |

Ключ:

```sql
PRIMARY KEY (collection_id, user_id)
```

Один пользователь может поставить одной коллекции только одну оценку.

## `chats`

Хранит только личные диалоги между двумя пользователями.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `UUID` | Уникальный идентификатор личного диалога |
| `first_user_id` | `UUID` | Первый участник |
| `second_user_id` | `UUID` | Второй участник |
| `first_user_last_read_at` | `TIMESTAMPTZ` | Время последнего прочтения первым участником |
| `second_user_last_read_at` | `TIMESTAMPTZ` | Время последнего прочтения вторым участником |
| `created_at` | `TIMESTAMPTZ` | Дата создания |
| `updated_at` | `TIMESTAMPTZ` | Дата обновления |

Ограничения:

- `CHECK (first_user_id <> second_user_id)` запрещает диалог пользователя с самим собой;
- уникальный индекс `ux_chats_private_pair` не позволяет создать два одинаковых личных диалога между одной и той же парой пользователей.

Например, диалог `A → B` и диалог `B → A` считаются одним и тем же диалогом.

## `messages`

Хранит сообщения в личных диалогах.

| Поле | Тип | Описание |
|---|---|---|
| `id` | `UUID` | Уникальный идентификатор сообщения |
| `chat_id` | `UUID` | Личный диалог |
| `sender_id` | `UUID` | Отправитель |
| `body` | `VARCHAR(2000)` | Текст сообщения |
| `is_deleted` | `BOOLEAN` | Признак удаления |
| `created_at` | `TIMESTAMPTZ` | Дата отправки |
| `updated_at` | `TIMESTAMPTZ` | Дата редактирования |

Дополнительная проверка:

- триггер `trg_messages_sender_is_participant` проверяет, что отправитель сообщения является участником выбранного личного диалога.

## 5. Представления

## `collection_stats`

Агрегирует статистику коллекции.

| Поле | Описание |
|---|---|
| `collection_id` | Коллекция |
| `item_count` | Количество предметов |
| `total_price_amount` | Общая стоимость предметов |
| `avg_rating` | Средняя оценка |
| `rating_count` | Количество оценок |
| `comment_count` | Количество комментариев |

Используется на странице коллекции.

---

## `user_profile_stats`

Агрегирует статистику профиля.

| Поле | Описание |
|---|---|
| `user_id` | Пользователь |
| `collection_count` | Количество коллекций |
| `item_count` | Количество предметов |
| `item_likes_count` | Количество лайков на предметах пользователя |
| `followers_count` | Количество подписчиков |
| `following_count` | Количество подписок пользователя |
| `total_collections_price` | Общая стоимость предметов во всех коллекциях пользователя |
| `avg_collection_rating` | Средняя оценка коллекций пользователя |

Используется в шапке профиля и на вкладке статистики.

---

## `chat_list`

Формирует список личных чатов с последним сообщением.

| Поле | Описание |
|---|---|
| `chat_id` | Диалог |
| `first_user_id` | Первый участник |
| `first_username` | Логин первого участника |
| `first_display_name` | Имя первого участника |
| `second_user_id` | Второй участник |
| `second_username` | Логин второго участника |
| `second_display_name` | Имя второго участника |
| `last_message_id` | Последнее сообщение |
| `last_message_sender_id` | Отправитель последнего сообщения |
| `last_message_body` | Текст последнего сообщения |
| `last_message_created_at` | Дата последнего сообщения |

Используется для бокового списка переписок.
